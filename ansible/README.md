# Ansible deployment

Deploy openBalena core + the extensions in this repo to a host. Idempotent; run individual
components with tags.

## Prerequisites
- A target host (Ubuntu 22.04/24.04) reachable over SSH, with sudo.
- `ansible` on your workstation; the `community.docker` collection:
  `ansible-galaxy collection install community.docker`
- DNS: a wildcard `*.${public_tld}` → the host IP (covers `api.`, `vpn.`, `cloudlink.`,
  `registry2.`, `delta.`, `builder.`, …). See [../docs/dns-tld-split.md](../docs/dns-tld-split.md).

## Use
```bash
cd ansible
cp inventory.example.ini inventory.ini            # set your host
cp group_vars/all.example.yml group_vars/all.yml  # set dns_tld, public_tld, secrets
ansible-vault encrypt group_vars/all.yml          # recommended: encrypt the secrets

ansible-playbook -i inventory.ini site.yml                       # everything
ansible-playbook -i inventory.ini site.yml --tags builder        # just the builder
ansible-playbook -i inventory.ini site.yml --tags imagemaker
```

## Roles & tags
| Role | Tag | Does |
|---|---|---|
| `core` | `core` | Clone + `make up` upstream open-balena with your `.env` |
| `haproxy` | `haproxy` | Install the custom `haproxy.cfg`, validate, reload |
| `builder` | `builder` | Build the patched image, lay down the override, start builder + dind |
| `imagemaker` | `imagemaker` | Install the imagemaker service + scripts |
| `admin` | `admin` | Bring up the community dashboard project |
| `delta` | `delta` | Bring up the delta project |

## Notes
- The `builder` role reads `TOKEN_AUTH_BUILDER_TOKEN` from the running API automatically (it's
  generated by core), so you don't have to copy it by hand — but you can also set it explicitly.
- Secrets live in `group_vars/all.yml` (vault-encrypt it). Nothing secret is committed.
